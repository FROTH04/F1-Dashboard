# 🏎 F1 Race Outcome Prediction — ML Project 2026

**THM Machine Learning Course Project**  
*Supervised by Prof. Nils Gumpfer (XAI Research, Explainable AI for Time Series)*

A full machine-learning pipeline that predicts Formula 1 race outcomes and championship standings for the 2026 season, with a strong focus on **Explainable AI (XAI)** — including SHAP for tree-based models and **LRP-SIGN for LSTM time series** (the professor's research specialty).

---

## What this project does

- **Predicts finishing positions** for all 22 drivers in each race using Random Forest and XGBoost
- **Classifies DNF probability** with a Gradient Boosting classifier
- **Models season trends** with an LSTM over the race sequence
- **Explains every prediction** with SHAP (TreeExplainer) and LRP-SIGN (implemented from scratch)
- **Simulates the full championship** with 1000 Monte Carlo runs → win probabilities + confidence intervals
- **Visualises everything** in an interactive Dash dashboard with a dedicated Predictions tab

```bash

```

### 2. Download real F1 data (optional — demo JSON included)
```bash

```

### 3. Train ML models
```bash
python ml_models.py          # uses FastF1 if available
```

### 4. Run XAI analyses
```bash
python shap_analysis.py --driver ANT --round 9 --save-plots
python lrp_timeseries.py --driver ANT --save-plots
```

### 5. Simulate the championship
```bash
python championship_simulator.py --simulations 1000
```

```bash
python f1_dashboard.py
```

## Key design decisions

**Strict temporal split** — training always uses earlier races, testing uses later races. No shuffle. This prevents data leakage (a fundamental requirement for time series).

**Demo fallback strategy** — all scripts work with `f1_data_2026.json` if FastF1 API is unavailable (e.g. at a presentation without internet). Real data is preferred when available.

**LRP-SIGN from scratch** — the LSTM explainability module implements the SIGN stabiliser variant of Layer-wise Relevance Propagation (Montavon et al., 2019) in pure NumPy. This answers: *which past race influenced the prediction most?*

**Clever-Hans check** — `shap_analysis.py` verifies that the most important SHAP features make domain sense (team strength + pace delta should dominate, not spurious correlates like race round number).

## News Context Layer

Performance updates are extracted from The Race
(the-race.com) and stored in `context_updates.json`.
This file is updated before each race weekend.

Data source: The Race (wearetherace.com) — independent
F1 journalism. Used for structured context extraction only.

To update context (requires personal Anthropic API key):

```bash
cp .env.example .env
# Add your ANTHROPIC_API_KEY to .env
python news_context.py
```

Users without an API key automatically use the
pre-processed `context_updates.json` included in the repo.
